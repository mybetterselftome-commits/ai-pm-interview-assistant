# AI PM 求职证据工作台 V4 PRD

## 一句话定位

AI PM 求职证据工作台帮助转行者把 JD 要求、过往经历、作品集方案、AI 知识和面试回答，转化成 AI 产品岗位认可的能力证据。

## 用户痛点

1. 看不懂 JD 背后的真实能力要求。
2. 不知道过往经历怎么转译成产品能力。
3. AI 产品知识点零散，不知道掌握到什么程度。
4. 不知道作品集该做什么、做到什么程度。
5. 面试回答像学习笔记，不像岗位匹配证明。
6. 准备过程碎片化，缺少复盘闭环。

## V4 核心主线

```text
JD 解码 → 经历转译 → 作品集规划 → 面试训练 → 补强闭环/我的资产
```

## 模块设计

### 1. JD 解码

目标：把岗位职责和任职要求拆成证据标准。

输出：
- JD 原文解码表
- 岗位能力地图
- 证据标准
- 高风险追问
- 作品集可补证据

### 2. 经历转译

目标：把用户过往运营、内容、项目、业务经历翻译成 AI PM 能力证据。

输出：
- 可用证据总览
- 经历到 AI PM 能力映射
- 简历 bullet
- 面试故事素材
- 不可夸大的边界

### 3. 作品集规划

目标：设计一个 7-14 天内可完成、能补齐岗位证据的作品集项目。

输出：
- 作品集定位
- MVP 范围
- AI 能力设计
- 指标体系
- Demo 演示脚本
- 面试讲法

### 4. 面试训练

目标：把学习笔记式回答改造成岗位证明式回答。

输出：
- 回答主张拆解
- 证据可信度评分
- 证据断点
- 抗追问压力测试
- 30 秒 / 90 秒回答素材
- 短板标签

### 5. 知识掌握度

目标：判断知识点是否达到面试可用、产品可用、抗追问可用的程度。

输出：
- 掌握度等级
- Rubric 评分
- 用户解释诊断
- 合格表达
- 抗追问版表达
- 补强任务

### 6. 补强闭环/我的资产

目标：把已有资产、短板标签和反馈记录整合为下一轮行动计划。

输出：
- 资产盘点
- 证据缺口排序
- 本轮补强任务
- 7 天闭环计划
- 复测机制

## 数据闭环

当前不新增数据库表，复用已有：

- `career_assets`：保存 JD 证据地图、经历转译、作品集规划、面试复盘、知识掌握卡、闭环计划。
- `career_feedback`：记录用户反馈。
- `weakness_tags`：记录面试和复盘中暴露的短板。
- `interview_sessions`：记录面试训练历史。

## MVP 边界

V4 不是：

- 自动投递工具。
- 替用户编造项目经历的工具。
- 完整 RAG 知识库。
- 自动完成作品集工程实现的平台。

V4 是：

- 岗位证据拆解工具。
- 经历转译工具。
- 作品集规划工具。
- 面试证据训练工具。
- 资产沉淀和补强闭环工具。

## 成功指标

- 用户能生成至少 1 份 JD 证据地图。
- 用户能生成至少 1 份经历证据转译。
- 用户能生成至少 1 个作品集方案。
- 用户能通过面试训练发现 3 个以上短板标签。
- 用户能导出完整求职证据资产包。
